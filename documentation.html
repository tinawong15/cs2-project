Description

Our website will benefit a user that is interested in knowing the locations, days, financial aid, and hours of farmers' markets in New York City boroughs. A user will be able to browse based on a specific aspect, such as borough, and receive the names of the farmers markets applicable and their respective data.

motivating questions

  • What markets are open on certain days, for example, Thursday?
  • Which markets offer financial help?
  • What are all the markets that are in this borough?
  • How many markets are in Queens and open on Wednesdays?
  • What is the exact street address of a specific market?

features of note

Starting from our index, one could click the link to a filtered search. From there, one may choose a variety of searches. If one does not fill out a filter, the search will be perfectly fine because the filter function doesn't require it.

The caption for the table counts the number of results based on the amount of filtering. Therefore, it provides the summary statistic for any of the filters.

The caption also shows all of the user's filters. This was created by implementing a function, using the cgi field storage values.

The text input for a market name provides an example in the box. The user also does not need to input the full name of a market; for example, if the user puts only one word, all markets that include that word will show up as results.

A link back to the index is given at the bottom of the rendering of the backEnd.py. Therefore, a user is able to scroll to the bottom of the page and is still able to easily go back and restart his or her search.

Bugs

A bug in our filter is that the market name search is case-sensitive.

When no query string is given, the backEnd.py raises a keyError.

Because of the way the data is given, there are some that give a range of days open, such as 'Monday-Thursday'. The selector only takes into account those two days, and does not know that market is also open between those days.

The links from index to documentation, filter, and overview use clyde, rather than homer. These pages would be unable to be accessed using homer.
